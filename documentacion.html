🎯 Introducción

toDus-API es una librería Python moderna y completa que proporciona un cliente XMPP/HTTP para interactuar con la red de mensajería instantánea cubana ToDus. Con esta librería puedes crear bots interactivos, manejar grupos, enviar contenido multimedia y mucho más.

✨ Cobertura del protocolo: 100% de las funciones de la app oficial.

¿Qué puedes hacer?

💬 Mensajería Privada

Envía y recibe mensajes de texto, edita, elimina y gestiona confirmaciones de lectura.

👥 Gestión de Grupos

Crea, gestiona y participa en grupos MUC Light con control total de permisos.

📁 Archivos y Multimedia

Sube y descarga archivos, imágenes, videos, stickers y contactos de forma segura.

🔄 Queue de Mensajes

Cola persistente con reintentos automáticos, ideal para bots offline-first.

🎚️ Control Avanzado

Manejo de presencia, privacidad, bloques y ubicaciones en tiempo real.

⚡ Rendimiento

Arquitectura asíncrona con soporte para múltiples conexiones simultáneas.

Aquí tienes un flujo básico para empezar con toDus-API en menos de 5 minutos.

$ Terminal

# Enviar mensaje
>>> client.send_message("5387654321", "¡Hola desde toDus-API!")

3. Escuchar mensajes entrantes

$ python3 listener.py
>>> def callback(msg):
... print(f"Mensaje: {msg.get('body')}")
>>> client.listen_messages(callback)

💡 Ejemplos Prácticos

Ejemplo 1: Bot Simple que Responde

$ bot_responder.py
# #!/usr/bin/env python3
from todus import ToDusClient2
client = ToDusClient2("5312345678", "password")
client.login()
def on_message(msg):
if msg.get('body'):
sender = msg.get('from')
client.send_message(sender, "Recibido: " + msg['body'])
client.listen_messages(on_message)

Ejemplo 2: Enviar Imagen

$ enviar_imagen.py
from todus import ToDusClient2, FileType
client = ToDusClient2("5312345678", "password")
client.login()
# Subir imagen
with open("foto.jpg", "rb") as f:
url = client.upload_file(f.read(), FileType.PICTURE)
# Enviar como mensaje
client.send_image_message("5387654321", url, "foto.jpg", 45678)

Ejemplo 3: Gestionar Grupos

$ gestor_grupos.py
from todus import ToDusClient2
client = ToDusClient2("5312345678", "password")
client.login()
# Unirse a grupo
client.groups.join("grupo_id_123")
# Enviar mensaje al grupo
client.send_message("grupo_id_123", "¡Hola grupo!")
# Salir del grupo
client.groups.leave("grupo_id_123")

✨ Características Principales

Autenticación

Método Descripción Uso
request_code(phone) Solicita código SMS client.request_code("5312345678")
validate_code(phone, code) Valida código SMS client.validate_code("5312345678", "1234")
login(phone, password) Autentica con credentials client.login("5312345678", "pass")

Estados de Mensaje

Los mensajes pueden estar en diferentes estados:

  • pending - Esperando envío
  • sent - Enviado al servidor
  • delivered - Entregado al destinatario
  • read - Leído por el destinatario
  • failed - Falló permanentemente

Tipos de Archivo

Tipo Código Ejemplo
Archivo FileType.FILE Documentos, PDFs, etc.
Voz FileType.VOICE Audios cortos
Audio FileType.AUDIO Canciones, música
Video FileType.VIDEO Videos, películas
Imagen FileType.PICTURE Fotos, imágenes

🔄 Message Queue - Bots Offline-First

El sistema de Message Queue permite que tu bot almacene mensajes localmente y los reintente automáticamente si fallan o hay desconexiones. Ideal para aplicaciones críticas.

Características

  • Almacenamiento persistente en SQLite
  • Reintentos automáticos con backoff exponencial
  • Callbacks para eventos de envío
  • Thread-safe y optimizado
  • Estadísticas en tiempo real

Uso Básico

$ bot_con_queue.py
from todus import ToDusClientWithQueue
client = ToDusClientWithQueue("5312345678", "password")
client.login()
# Registrar callback
def on_delivered(msg):
print(f"✓ Entregado: {msg.msg_id}")
client.register_on_message_delivered(on_delivered)
# Enviar con queue
client.send_message_queued("5312345678", "Hola con retry automático")

Ver Estadísticas

$ Estadísticas de la cola
stats = client.get_queue_stats()
print(stats)
{'pending': 2, 'sent': 5, 'delivered': 12, 'read': 8, 'failed': 1}

Backoff Exponencial

Los reintentos se hacen con delays progresivos:

Intento Fórmula Delay
1 2^0 ~1s
2 2^1 ~2s
3 2^2 ~4s
N 2^N (máx 300s) Exponencial capped
💡 Tip: El queue se persiste en ~/.todus/messages.db, así que sobrevive a reinicios del programa.

❓ Preguntas Frecuentes

¿Necesito ser desarrollador para usar esto?

No necesariamente. Si conoces Python básico, puedes crear bots simples. Para cosas más avanzadas, se recomienda tener conocimientos de programación.

¿Es seguro para producción?

Sí, el código está optimizado, tiene tests y es usado en múltiples proyectos en producción. Asegúrate de manejar credenciales de forma segura (variables de entorno, no hardcodear).
